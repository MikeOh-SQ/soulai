# ADHDQQ.COM 발표자료

## 1. 제목

ADHDQQ.COM: 자가보고와 반응성 테스트를 결합한 성인 ADHD 선별 보조 모바일 웹 앱

발표 핵심 문장:

- 이 서비스는 진단을 확정하지 않습니다.
- 사용자가 느끼는 어려움과 실제 수행에서 보이는 반응을 함께 보여줍니다.
- 결과를 바탕으로 바로 실천 가능한 작은 행동 계획까지 연결합니다.

## 2. 문제 정의

성인 ADHD 관련 어려움은 질문지만으로 설명하기 어렵습니다.

- 사람마다 자기 인식이 다릅니다.
- 피로, 긴장, 수면, 환경에 따라 수행이 달라집니다.
- 부주의, 충동성, 활동성은 한 가지 방식으로만 측정하기 어렵습니다.
- 사용자는 결과를 받아도 "그래서 오늘 무엇을 해야 하는지"를 알기 어렵습니다.

따라서 이 서비스는 설문, 수행 과제, 실행 계획을 하나의 흐름으로 묶었습니다.

## 3. 서비스 전체 흐름

1. `ID`: 사용자 기록 생성 또는 불러오기
2. `SELF`: 자가보고 선별 6문항
3. `SYM`: 증상 기준 체크 23문항
4. `REACT`: 반응성 테스트 3종
5. `RESULT`: 통합 리포트
6. `PLAN`: 유형별 실행 계획과 AI 조정 채팅

하단 네비게이션은 `ID / SELF / SYM / REACT / RESULT / PLAN`으로 표시합니다.

## 4. 하이브리드 작동 원리

이 서비스는 "AI가 전부 판단하는 구조"가 아닙니다.

### 1단계: 브라우저 측정

- 사용자의 문항 응답을 수집합니다.
- 반응성 테스트의 시행별 데이터를 저장합니다.
- 터치, 클릭, 센서, 롱터치 입력을 기록합니다.

### 2단계: 서버 결정론적 계산

- 저장 JSON의 숫자를 읽습니다.
- 부주의, 충동성, 활동성 관련 지표를 계산합니다.
- 통합 리포트 본문은 서버 계산값을 우선 사용합니다.
- JSON에 없는 값은 추정하지 않습니다.

### 3단계: AI 보조

- 자가보고 선별의 짧은 설명
- 증상 기준 체크의 짧은 설명
- 실행 계획 문장
- 사용자의 생활 패턴에 맞춘 plan 조정 채팅

AI는 설명과 코칭을 돕지만, 핵심 수치 계산과 리포트 구조는 서버 규칙이 담당합니다.

## 5. 왜 이런 구조인가

자가보고는 사용자의 체감 어려움을 잘 보여줍니다. 반면 반응성 테스트는 짧은 과제 상황에서 실제 수행 패턴을 보여줍니다.

두 결과는 항상 일치하지 않습니다.

- 평소에는 괜찮다고 느끼지만 과제에서는 목표를 놓칠 수 있습니다.
- 평소에는 힘들다고 느끼지만 검사 상황에서는 안정적으로 수행할 수 있습니다.
- 피로, 수면, 긴장, 검사 환경에 따라 결과가 달라질 수 있습니다.

그래서 이 앱은 주관 결과와 객관 수행 결과를 교차 확인합니다.

## 6. SELF: 자가보고 선별

목적:

- 최근 6개월 동안의 체감 어려움을 빠르게 확인합니다.
- 부주의와 과잉행동·충동성 신호를 나눠 봅니다.

차용 내용:

- 성인 ADHD 자기보고 선별 도구 연구에서 6문항 기반 빠른 스크리닝 구조를 참고했습니다.
- 사용자 화면에서는 저작권/상표 노출을 줄이기 위해 `자가보고 선별`이라는 일반 명칭을 사용합니다.

저장 지표:

- `attention_score`
- `hyperactivity_score`
- `positive_count`

## 7. SYM: 증상 기준 체크

목적:

- 부주의와 과잉행동·충동성 신호가 어느 쪽에 더 모이는지 확인합니다.
- 추가 확인 문항으로 지속성, 다환경성, 기능 손상 맥락을 보강합니다.

차용 내용:

- ADHD 증상 기준의 부주의 9문항, 과잉행동/충동성 9문항 구분을 참고했습니다.
- 사용자 화면에서는 `증상 기준 체크`라는 일반 명칭을 사용합니다.

저장 지표:

- `inattention_true_count`
- `hyperactivity_true_count`
- `contextual_true_count`
- `subtype`

## 8. REACT 1: 신호 찾기

목적:

- 지속주의와 목표 놓침을 봅니다.
- 반응시간이 안정적인지 봅니다.

방식:

- 목표 자극에만 반응합니다.
- 비목표 자극에는 반응하지 않습니다.
- 60회 본 시행을 진행합니다.
- ISI를 1000/1250/1500ms로 섞어 예측 반응을 줄입니다.

핵심 지표:

- `omission_rate`: 목표 놓침 비율
- `reaction_time_variability`: 반응시간 변동성
- `tau`: 유난히 느린 반응 꼬리
- `late_phase_drop`: 후반부 정확도 저하

근거:

- 지속수행검사 계열은 목표 놓침과 반응시간을 주의 수행 지표로 활용합니다.
- Tamm et al.과 Kofler et al.은 ADHD에서 반응시간 변동성이 중요한 연구 주제임을 정리했습니다.

## 9. REACT 2: 멈춤 버튼

목적:

- 눌러야 할 때는 누르고, 멈춰야 할 때는 멈추는지 봅니다.
- 성급한 반응과 억제 실패를 분리해 봅니다.

방식:

- `o.gif`는 누릅니다.
- `x.gif`는 누르지 않습니다.
- 본 시행 50회입니다.
- Go 자극 시작 후 220ms 이후부터 성공으로 인정합니다.
- No-Go는 최소 2회 이상 Go 뒤에 나오도록 배치합니다.
- iPhone/iPad는 시각/터치 지연을 고려해 380ms 보정합니다.

핵심 지표:

- `commission_rate`: 누르면 안 되는 자극을 누른 비율
- `fast_error_rate`: 너무 빠른 반응 비율
- `premature_response_count`: 성공 기준보다 빠른 Go 반응
- `mean_go_reaction_time`: Go 평균 반응시간

근거:

- Go/No-Go 과제는 반응 억제와 commission error를 보는 대표적인 수행 과제 구조입니다.
- 이 앱은 단순히 틀린 횟수만 보지 않고 반응시간과 함께 패턴을 해석합니다.

## 10. REACT 3: 균형 유지

목적:

- 활동성, 안절부절함, 안정 유지 능력을 보조적으로 봅니다.

방식:

- 모바일 센서가 가능하면 센서를 사용합니다.
- 센서가 불가능하면 중앙 원 롱터치 방식으로 진행합니다.
- PC에서는 센서가 있는 것처럼 감지되는 오류를 막고 포인터 fallback으로 진행합니다.

핵심 지표:

- `stable_duration_pct`: 안정 유지 비율
- `spike_count`: 큰 흔들림 횟수
- `total_movement`: 전체 움직임 크기
- `input_source`: 입력 방식

근거:

- QbCheck/QbTest 계열은 수행 중 오류, 반응시간, 활동성을 함께 보는 구조를 사용합니다.
- 다만 성인 ADHD 임상 표본에서 객관 테스트 해석에는 주의가 필요하다는 연구도 있어, 이 앱은 활동성 지표를 보조 참고값으로만 사용합니다.

## 11. RESULT: 통합 리포트

리포트는 세 영역을 함께 보여줍니다.

- 주관적 보고: 사용자가 느끼는 어려움
- 객관적 반응: 테스트에서 보인 수행 패턴
- 일치 여부: 두 결과가 같은 방향인지 다른 방향인지

부주의 탭:

- 목표 놓침 비율
- 반응시간 변동성
- Tau
- 후반부 정확도 저하

충동성 탭:

- 잘못된 반응 비율
- 성급 반응 비율
- Go 반응시간
- 반응 패턴

## 12. PLAN: 실행 계획

결과만 보여주면 사용자가 다음 행동을 정하기 어렵습니다. 그래서 마지막 단계에서 바로 실천할 수 있는 3개 계획을 제공합니다.

분기:

- 부주의 경향: 과제 분할, 시각적 단서, 환경 통제
- 충동성 경향: 지연 행동, 자기 점검, 심호흡
- 복합형 경향: 두 전략을 함께 사용
- 안정 범위: 현재 루틴 유지

근거:

- 성인 ADHD CBT 연구는 시간관리, 조직화, 계획, 자기점검 전략의 유용성을 보고합니다.
- 이 앱은 이를 의료적 처방이 아니라 일상 코칭 문장으로 바꿉니다.

## 13. 서비스의 차별점

- 질문지만 보는 구조가 아닙니다.
- 게임 점수만 보여주는 구조도 아닙니다.
- 저장된 수치 기반 리포트와 AI 코칭을 분리했습니다.
- 모바일 웹에서 센서가 되지 않는 경우까지 fallback을 설계했습니다.
- 결과에서 바로 실행 계획까지 연결합니다.

## 14. 한계

- 진단 확정 도구가 아닙니다.
- 반응성 테스트는 환경, 기기 성능, 터치 지연, 피로의 영향을 받습니다.
- 균형 유지 테스트는 과잉행동을 직접 대체하지 못합니다.
- 객관 수행 테스트는 보조 지표이며, 필요 시 전문가 평가와 함께 봐야 합니다.

## 15. 향후 개선

- 더 많은 사용자 데이터 기반 기준값 보정
- 기기별 터치/시각 지연 보정 고도화
- 반응성 테스트 validity 기준 개선
- report와 plan 문장의 임상적 안전성 검토
- 관리자용 데이터 검토 화면 분리
