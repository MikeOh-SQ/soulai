# plan 프롬프트 작성 설명서

이 문서는 `plan` 단계에 넣을 AI 프롬프트를 어떻게 구성하면 좋은지 쉽게 설명하는 가이드입니다.

이 프로젝트의 `plan` 화면은 크게 2가지를 보여줍니다.

1. `plan.suggestions`
사용자가 바로 해볼 수 있는 실행 제안 3개

2. `plan.openingMessage`
사용자가 "이 계획을 내 생활에 맞게 바꿔 달라"고 이어서 말할 수 있게 여는 짧은 안내 문구


## 1. plan 프롬프트의 목적

plan 프롬프트는 "결과를 설명하는 것"보다 "사용자가 바로 실천할 수 있게 바꾸는 것"이 더 중요합니다.

좋은 plan 프롬프트는 아래를 해야 합니다.

- 리포트와 검사 결과를 바탕으로 한다
- 사용자가 오늘 바로 시작할 수 있는 행동으로 바꾼다
- 너무 일반적인 조언 대신 구체적인 행동을 제안한다
- 진단처럼 말하지 않는다
- 사용자가 이후 채팅으로 계획을 수정하기 쉽게 만든다


## 2. plan 프롬프트에 꼭 들어가야 하는 정보

프롬프트에는 아래 정보가 들어가면 좋습니다.

- 서비스 역할
- AI의 역할
- 말투와 금지사항
- 참고할 입력 데이터
- 출력 형식
- 각 항목 작성 규칙


## 3. 가장 쉬운 프롬프트 구조

아래 순서대로 쓰면 됩니다.

### 1) 서비스 설명

예:

`서비스: ADHD 선별/감별 보조 모바일 웹 앱의 실행 계획 생성 단계`


### 2) AI 역할 정의

예:

`역할: 검사 결과를 바탕으로 사용자가 바로 실천할 수 있는 계획을 만드는 한국어 행동 코치`


### 3) 중요한 금지사항

여기는 짧고 분명해야 합니다.

예:

- 진단 확정처럼 말하지 말 것
- 의료적 처방처럼 말하지 말 것
- 추상적인 위로만 하지 말 것
- 계획은 짧고 구체적으로 작성할 것


### 4) 참고 데이터 넣기

AI가 보고 계획을 만들 근거를 적습니다.

예:

- 기록 JSON
- 핵심 지표
- report 요약
- ASRS / DSM-5 / 반응성 테스트 결과


### 5) 출력 형식 지정

이 부분이 가장 중요합니다.
AI가 무엇을 몇 개 만들지 명확히 써야 합니다.

예:

- `plan.suggestions`는 3개
- 각 제안은 한 문단 또는 한 문장
- `plan.openingMessage`는 1~2문장


### 6) 각 제안 작성 규칙

좋은 실행 계획은 아래 조건을 붙이면 품질이 좋아집니다.

- 바로 실행 가능한 행동일 것
- 시간, 장소, 행동 단위가 드러날 것
- 너무 많은 일을 한 번에 시키지 말 것
- 실패 가능성을 낮추는 방향일 것
- 사용자의 부주의/충동성 경향 중 어떤 문제를 줄이려는지 암묵적으로 반영할 것


## 4. 좋은 제안의 기준

좋은 plan 제안은 이런 느낌입니다.

- "아침에 해야 할 일을 머리로만 기억하지 말고, 출근 전 확인할 체크리스트를 휴대폰 첫 화면에 3개만 고정해 두세요."
- "집중이 가장 잘 되는 시간대에 20분짜리 시작 블록을 먼저 넣고, 끝나면 5분 쉬는 식으로 하루 첫 과제를 시작해 보세요."
- "자주 미루는 일은 '끝내기'보다 '5분만 시작하기' 기준으로 바꿔서 진입 장벽을 낮추세요."

이런 제안은 좋지 않습니다.

- "더 열심히 해보세요."
- "생활 습관을 잘 관리하세요."
- "집중력을 높이는 노력이 필요합니다."

이유:
너무 추상적이라 바로 행동으로 옮기기 어렵기 때문입니다.


## 5. openingMessage는 어떻게 써야 하나

openingMessage의 목적은 사용자가 plan 채팅을 시작하게 만드는 것입니다.

즉, 설명 문구가 아니라 "수정 요청을 유도하는 문구"여야 합니다.

좋은 예:

- "생활 패턴이나 일정에 맞춰 계획을 더 현실적으로 바꿀 수 있어요. 출근 시간이나 자주 무너지는 시간을 알려 주세요."
- "지금 제안이 너무 어렵거나 안 맞으면, 수면 시간이나 업무 패턴에 맞게 다시 조정해 드릴게요."

좋지 않은 예:

- "아래 계획을 참고하세요."
- "다음은 추천 계획입니다."

이유:
대화를 열어 주지 못하기 때문입니다.


## 6. 추천 프롬프트 틀

아래 틀을 그대로 시작점으로 써도 됩니다.

```txt
서비스: ADHD 선별/감별 보조 모바일 웹 앱의 실행 계획 생성 단계
역할: 검사 결과를 바탕으로 사용자가 바로 실천할 수 있는 계획을 만드는 한국어 행동 코치

중요:
- 진단 확정처럼 말하지 말 것
- 의료적 처방처럼 말하지 말 것
- 추상적인 조언보다 바로 실행 가능한 행동을 제안할 것
- 사용자가 부담 없이 시작할 수 있도록 현실적인 수준으로 작성할 것

참고 데이터:
- 대상 기록 JSON
- 계산된 핵심 지표
- report 요약

요청:
1. plan.suggestions는 3개 작성
2. 각 suggestion은 한국어로 짧고 구체적으로 작성
3. 각 suggestion은 오늘 바로 시작 가능한 행동이어야 함
4. 시간, 장소, 행동 단위 중 최소 1개 이상이 드러나야 함
5. 비슷한 내용 반복 금지
6. plan.openingMessage는 1~2문장으로 작성
7. openingMessage는 사용자가 생활 패턴에 맞게 계획 수정을 요청하도록 유도할 것
```


## 7. 더 잘 만들고 싶을 때 추가하면 좋은 조건

필요하면 아래 조건을 더 붙일 수 있습니다.

- 부주의가 높으면: 시작 장벽 낮추기, 외부 단서 만들기, 할 일 쪼개기 중심
- 충동성이 높으면: 멈춤 장치, 알림 끄기, 즉시 반응 줄이기 중심
- 생활 피로도가 높으면: 계획 개수를 줄이고 회복 시간을 먼저 확보
- 직장인/학생/야간형 등 상황이 보이면 그 맥락 반영


## 8. 최종 점검 체크리스트

plan 프롬프트를 만들고 나면 아래만 확인하면 됩니다.

- 제안이 3개인가
- 각 제안이 바로 행동으로 옮길 수 있는가
- 진단처럼 말하지 않는가
- 너무 추상적이지 않은가
- openingMessage가 사용자의 추가 입력을 유도하는가


## 9. 한 줄 요약

좋은 plan 프롬프트는 "결과 설명"이 아니라 "바로 실행할 수 있는 3가지 행동 + 사용자가 수정 요청을 시작하게 만드는 한 문장"을 만드는 프롬프트입니다.
