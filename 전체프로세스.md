# 전체 프로세스

이 문서는 앱의 각 단계에서 어떤 입력이 어떤 JSON 변수로 저장되고, 어떤 AI 프롬프트를 거쳐 어떤 화면 결과가 만들어지는지 코드 기준으로 정리한 문서입니다.

## 1. 시작과 ID 생성

### 사용자 입력

- 사용자는 `ID`를 새로 만들거나 기존 기록을 불러옵니다.

### 저장 구조

- 새 ID 생성 시 [public/app.js](/home/sqmini/soulai/public/app.js:163)의 `createEmptyRecord()`가 기본 레코드를 만듭니다.
- 기본 구조:

```json
{
  "id": "user-id",
  "createdAt": "ISO",
  "updatedAt": "ISO",
  "fileName": "user-id-YYYYMMDD-HHMMSS.json",
  "currentStep": "id",
  "tests": {
    "asrs": [],
    "dsm5": [],
    "game": {}
  },
  "report": null,
  "plan": {
    "suggestions": [],
    "chat": []
  }
}
```

### 파일 저장

- [public/app.js](/home/sqmini/soulai/public/app.js:528)의 `persistRecord()`가 `/api/records`로 보냅니다.
- [server.js](/home/sqmini/soulai/server.js:738) 이후의 `/api/records` POST가 실제 파일을 `database/*.json`으로 저장합니다.

## 2. ASRS

### 질문 소스

- 질문은 [config/asrs.json](/home/sqmini/soulai/config/asrs.json)에서 읽습니다.
- 프런트 로딩은 [public/app.js](/home/sqmini/soulai/public/app.js:110)의 `loadConfigs()`에서 처리합니다.

### 화면 표시

- ASRS 화면 렌더링은 [public/app.js](/home/sqmini/soulai/public/app.js:2378) 부근의 `pages.asrs()`가 담당합니다.
- 문항별 이미지는 현재 `public/asrs01.png` ~ `public/asrs06.png`를 사용합니다.

### 사용자 응답 → 메모리 저장

- 사용자가 0~4 버튼을 누르면 [public/app.js](/home/sqmini/soulai/public/app.js:2859) 이벤트가 [public/app.js](/home/sqmini/soulai/public/app.js:552)의 `answerAsrs(value)`를 호출합니다.
- 이 함수는 현재 문항 응답을 아래 구조로 `state.currentRecord.tests.asrs[index]`에 넣습니다.

```json
{
  "prompt": "질문 문장",
  "examples": ["예시1", "예시2"],
  "answer": 3
}
```

### 저장용 JSON 변환

- 파일 저장 시 [public/app.js](/home/sqmini/soulai/public/app.js:396)의 `summarizeAsrsForStorage()`가 아래 요약 구조로 바꿉니다.

```json
"asrs": {
  "responses": [3, 4, 2, 3, 4, 3],
  "attention_score": 12,
  "attention_max": 16,
  "hyperactivity_score": 7,
  "hyperactivity_max": 8,
  "total_score": 19,
  "total_max": 24,
  "positive_count": 6
}
```

### 로컬 계산 규칙

- [public/app.js](/home/sqmini/soulai/public/app.js:309)의 `analyzeAsrs()`가 다음 값을 계산합니다.
- `positive_count`
- `attentionPositive`
- `hyperPositive`
- `summary`
- `attentionMessage`
- `hyperMessage`
- `guidance`

### AI 사용 여부

- ASRS를 끝내면 [public/app.js](/home/sqmini/soulai/public/app.js:1719)의 `generateAsrsAnalysis()`가 `/api/ai/asrs-analysis`를 호출합니다.
- 프롬프트는 [server.js](/home/sqmini/soulai/server.js:566)의 `buildAsrsAnalysisPrompt()`에서 생성됩니다.
- 이 프롬프트에 들어가는 핵심 입력은:
  - 저장 전 현재 레코드 JSON
  - `analyzeAsrs()` 결과
  - `computeAssessmentMetrics(record)` 결과

### AI 출력 형식

- 서버는 Gemini에 아래 형식을 요구합니다.

```json
{
  "summary": "string",
  "attention": "string",
  "hyperactivity": "string",
  "guidance": "string"
}
```

### 화면 결과

- 저장 위치: `state.currentRecord.asrsAnalysis`
- 사용 위치: [public/app.js](/home/sqmini/soulai/public/app.js:2242) 부근의 `pages["asrs-result"]`
- 즉, ASRS 직후 보이는 간단 리포트는:
  - 점수와 양성 개수는 로컬 계산
  - 문장형 해석은 Gemini

## 3. DSM-5

- 다만 이후 report 생성 시 DSM 저장값이 핵심 입력으로 다시 사용됩니다.

## 6. report 생성

### 진입 조건

- 게임 완료 후 `report` 이동 시 [public/app.js](/home/sqmini/soulai/public/app.js:1792)의 `ensureInsights()`가 실행됩니다.
- `report.schemaVersion !== 2`이거나 report가 없으면 다시 생성합니다.

### 서버 입력

- [public/app.js](/home/sqmini/soulai/public/app.js:1690)의 `generateReportAndPlan()`이 `/api/ai/insights`로 현재 record 전체를 보냅니다.

### 서버 계산

- [server.js](/home/sqmini/soulai/server.js:230)의 `computeAssessmentMetrics(record)`가 report용 핵심 지표를 계산합니다.
- 여기서 사용되는 저장 JSON 변수:
  - `tests.asrs.responses`
  - `tests.dsm5.inattention_true_count`
  - `tests.dsm5.hyperactivity_true_count`
  - `tests.dsm5.contextual_true_count`
  - `tests.game.tests.signal_detection.*`
  - `tests.game.tests.go_nogo.*`
  - `tests.game.tests.balance_hold.*`

### 계산 결과 예시

- `severity`
- `asrsAttentionScore`
- `asrsImpulseScore`
- `omissionRate`
- `commissionRate`
- `reactionVariability`
- `subjectiveDomain`
- `objectiveDomain`
- `alignment`
- `dailyImpactLevel`
- `scores.attention/executive/impulse/emotion/structure`

### report 본문 생성 방식

- [server.js](/home/sqmini/soulai/server.js:323)의 `buildDeterministicReport(metrics)`가 report 문장을 만듭니다.
- 중요한 점:
  - report는 Gemini 응답을 직접 사용하지 않습니다.
  - 저장 JSON 수치와 서버 계산값만 사용합니다.
  - 세부 반응성 수치가 없는 오래된 JSON은 `score` 중심으로만 설명합니다.

### AI 프롬프트와의 관계

- [server.js](/home/sqmini/soulai/server.js:523)의 `buildInsightsPrompt()`는 여전히 만들어지지만,
- 현재 `generateInsights()`에서는 report 본문은 `buildDeterministicReport(metrics)` 결과를 최종 사용합니다.
- 즉:
  - `report`는 deterministic
  - `plan`은 Gemini가 있으면 Gemini 문장 사용 가능

### 최종 report JSON

- `report.schemaVersion = 2`
- `report.hero`
- `report.crossCheck`
- `report.profile`
- `report.dailyImpact`
- `report.sections`
- `report.bridge`

### 프런트 표시

- 화면 렌더링은 [public/app.js](/home/sqmini/soulai/public/app.js:2518)의 `pages.report()`
- 프런트는 [public/app.js](/home/sqmini/soulai/public/app.js:1820)의 `computeLocalReportViewModel()`로 저장 JSON을 다시 읽어 화면용 모델을 만듭니다.

## 7. plan 생성과 AI 채팅

### 초기 plan 생성

- `/api/ai/insights` 응답에는 `plan`도 함께 포함됩니다.
- [server.js](/home/sqmini/soulai/server.js:602)의 `generateInsights()`가:
  - Gemini 응답에 `plan.suggestions`가 있으면 그것을 사용
  - 없으면 서버 기본 suggestions를 사용
  - 첫 assistant 메시지는 `payload.plan.openingMessage` 또는 서버 기본 openingMessage를 사용

### 추가 채팅

- 사용자가 plan 화면에서 메시지를 보내면 [public/app.js](/home/sqmini/soulai/public/app.js:1919)의 `sendPlanChat()`이 `/api/ai/chat`을 호출합니다.
- 프롬프트 생성은 [server.js](/home/sqmini/soulai/server.js:550)의 `buildChatPrompt(record, message)`

입력:

- 현재 전체 record JSON
- 사용자가 plan 화면에 쓴 자유 텍스트

출력 형식:

```json
{
  "reply": "string",
  "additionalSuggestion": "string or empty string"
}
```

결과 반영:

- `plan.chat.push({ role: "assistant", text: reply })`
- `additionalSuggestion`이 있으면 최근 3개 suggestions에 반영

## 8. 오래된 파일과 새 스키마

- 과거 JSON은 반응성 테스트 상세 수치가 없을 수 있습니다.
- 이런 경우 report는 없는 수치를 추정하지 않고:
  - `signal_detection.score`
  - `go_nogo.score`
  - `balance_hold.score`
  중심 설명만 합니다.
- `report`나 `plan` 단계에서 오래된 파일을 불러오면 [public/app.js](/home/sqmini/soulai/public/app.js:652)의 `handleLoadRecord()`가 `ensureInsights()`를 다시 호출해 새 report 스키마를 생성합니다.

## 9. 한 줄 정리

- ASRS quick analysis: `로컬 점수 계산 + Gemini 문장`
- DSM-5: `로컬 집계`
- 반응성 테스트: `런타임 측정 -> 상세 수치 저장`
- report: `저장된 JSON 수치 + 서버 deterministic 계산`
- plan: `기본값은 서버 계산, 채팅은 Gemini`
