# 질문과 해석

이 문서는 현재 저장되는 JSON 값이 무엇을 뜻하는지와, 화면에서 보이는 ASRS, DSM-5, 반응성 테스트, report 결과를 어떻게 해석하는지를 정리한 문서입니다.

## 저장 JSON 원칙

- 저장 파일은 문항 전문과 장문 로그를 넣지 않고, 점수와 핵심 집계값 중심으로 저장합니다.
- report는 저장된 JSON 수치만 기준으로 생성됩니다.
- JSON에 없는 반응성 테스트 세부 수치는 report 문장에 인용하지 않습니다.
- `schemaVersion: 2` report부터 현재 리포트 규칙이 반영됩니다.

## ASRS

저장 예시:

```json
"asrs": {
  "responses": [1, 2, 3, 2, 4, 1],
  "attention_score": 8,
  "attention_max": 16,
  "hyperactivity_score": 5,
  "hyperactivity_max": 8,
  "total_score": 13,
  "total_max": 24,
  "positive_count": 4
}
```

값 설명:

- `responses`: 6개 문항의 원점수 배열
- `attention_score`: 1~4번 문항 합계
- `attention_max`: 1~4번 문항의 최대 가능 점수
- `hyperactivity_score`: 5~6번 문항 합계
- `hyperactivity_max`: 5~6번 문항의 최대 가능 점수
- `total_score`: 6문항 전체 합계
- `total_max`: 전체 최대 가능 점수
- `positive_count`: ASRS 기준상 유의미 문항 수

해석 규칙:

- 1~3번은 `2점 이상`이면 유의미
- 4~6번은 `3점 이상`이면 유의미
- `positive_count >= 4`면 선별 신호가 비교적 높은 편으로 봅니다
- `attention_score`가 높을수록 집중 유지, 시작 지연, 마감 관리 쪽 어려움 가능성이 커집니다
- `hyperactivity_score`가 높을수록 안절부절함, 끼어들기, 충동적 반응 가능성이 커집니다

ASRS quick analysis:

- AI가 생성하는 값은 `summary`, `attention`, `hyperactivity`, `guidance`입니다.
- 이 분석은 ASRS 완료 직후 보이는 간단 리포트에 사용됩니다.

## DSM-5

저장 예시:

```json
"dsm5": {
  "responses": [true, false, true, true, false, true, true, false, false, true, false, true, false, false, true, true, false, false, true, true, true, false, true],
  "inattention_true_count": 5,
  "hyperactivity_true_count": 4,
  "contextual_true_count": 4,
  "total_true_count": 9,
  "subtype": "판정 보류"
}
```

문항 구성:

- 1~9번: 부주의
- 10~18번: 과잉행동/충동성
- 19~23번: 추가 확인 문항

값 설명:

- `responses`: DSM-5 전체 문항의 Yes/No 배열
- `inattention_true_count`: 부주의 9문항 중 Yes 개수
- `hyperactivity_true_count`: 과잉행동/충동성 9문항 중 Yes 개수
- `contextual_true_count`: 추가 확인 5문항 중 Yes 개수
- `total_true_count`: 부주의 + 과잉행동/충동성 Yes 총합
- `subtype`: 현재 규칙으로 계산한 유형 문자열

해석 규칙:

- `inattention_true_count >= 6` and `hyperactivity_true_count < 6`
  - `부주의형 가능성`
- `hyperactivity_true_count >= 6` and `inattention_true_count < 6`
  - `과잉행동/충동형 가능성`
- 두 값이 모두 `6 이상`
  - `복합형 가능성`
- 두 값이 모두 `6 미만`
  - `무증상 범위`
- 전체 문항 응답이 끝나지 않으면
  - `판정 보류`

추가 확인 문항 해석:

- `contextual_true_count`는 subtype 점수를 직접 만들지는 않지만 report의 일상 피로도와 맥락 해석을 보강합니다.
- 값이 높을수록 지속성, 조기 발현, 다환경성, 기능 손상, 감별 필요성 조건이 더 많이 맞는다는 뜻입니다.

## 반응성 테스트

저장 예시:

```json
"game": {
  "status": "completed",
  "currentTestIndex": 2,
  "currentTestKey": "balance_hold",
  "tests": {
    "signal_detection": {
      "score": 81,
      "target_count": 8,
      "hit_count": 7,
      "omission_errors": 1,
      "false_alarm_count": 0,
      "omission_rate": 0.13,
      "mean_reaction_time": 702,
      "reaction_time_variability": 142,
      "late_response_count": 0
    },
    "go_nogo": {
      "score": 67,
      "go_count": 35,
      "nogo_count": 15,
      "go_hit_count": 30,
      "go_omission_count": 5,
      "commission_errors": 3,
      "successful_stop_count": 12,
      "premature_response_count": 2,
      "inhibition_failure_rate": 0.2
    },
    "balance_hold": {
      "score": 72,
      "stable_hold_time": 22.4,
      "movement_variability": 0.31,
      "large_motion_count": 3,
      "correction_count": 12,
      "inside_target_ratio": 0.74
    }
  },
  "summary": {
    "inattention_signal": {
      "score": 81,
      "source_test": "signal_detection"
    },
    "impulsivity_signal": {
      "score": 67,
      "source_test": "go_nogo"
    },
    "activity_signal": {
      "score": 72,
      "source_test": "balance_hold"
    },
    "summary": "충동성 관련 신호가 상대적으로 더 높게 관찰됩니다."
  }
}
```

공통 값 설명:

- `status`: `pending`, `running`, `completed`
- `currentTestIndex`: 현재 진행 중이거나 마지막으로 완료한 테스트 순번
- `currentTestKey`: 현재 테스트 키
- `summary`: 3개 테스트를 합친 요약 구조

### signal_detection

- `score`: 지속주의/표적 탐지 점수
- `target_count`: 목표 자극 수
- `hit_count`: 올바르게 반응한 수
- `omission_errors`: 목표를 놓친 수
- `false_alarm_count`: 잘못 누른 수
- `omission_rate`: 목표 놓침 비율
- `mean_reaction_time`: 평균 반응시간
- `reaction_time_variability`: 반응시간 변동성
- `late_response_count`: 늦은 반응 수

해석:

- 점수가 낮거나 `omission_rate`, `reaction_time_variability`가 높으면 부주의 신호가 상대적으로 큽니다.

### go_nogo

- `score`: 반응 억제/충동 조절 점수
- `go_count`: 눌러야 하는 trial 수
- `nogo_count`: 누르면 안 되는 trial 수
- `go_hit_count`: 올바른 go 반응 수
- `go_omission_count`: go를 놓친 수
- `commission_errors`: 누르면 안 되는 자극에 반응한 수
- `successful_stop_count`: no-go 성공 수
- `false_stop_count`: 불필요하게 멈춘 수
- `late_stop_count`: 늦은 반응 수
- `mean_go_reaction_time`: go 평균 반응시간
- `premature_response_count`: 너무 이른 반응 수
- `inhibition_failure_rate`: 잘못된 반응 비율
- `hold_success_rate`: no-go 유지 성공률

해석:

- 점수가 낮거나 `commission_errors`, `premature_response_count`, `inhibition_failure_rate`가 높으면 충동성 또는 억제 조절 어려움 신호가 큽니다.

### balance_hold

- `score`: 활동성/자기조절 점수
- `stable_hold_time`: 안정 유지 시간
- `drift_distance`: 중심에서 벗어난 거리 누적
- `movement_variability`: 움직임 변동성
- `large_motion_count`: 큰 흔들림 횟수
- `correction_count`: 보정 횟수
- `inside_target_ratio`: 목표 영역 안에 머문 비율
- `input_source`: `device-motion` 또는 `pointer`

해석:

- 점수가 낮거나 `movement_variability`, `large_motion_count`가 높으면 활동성/자기조절의 어려움 신호가 큽니다.

## report

저장 예시:

```json
"report": {
  "schemaVersion": 2,
  "generatedAt": "2026-04-13T05:59:57.113Z",
  "severity": "높음",
  "scores": {
    "attention": 20,
    "executive": 18,
    "impulse": 20,
    "emotion": 18,
    "structure": 21
  },
  "hero": {
    "badges": ["#주의력충전필요", "#반응속도조절중"],
    "summary": "..."
  },
  "crossCheck": {
    "subjectiveTitle": "...",
    "subjectiveText": "...",
    "objectiveTitle": "...",
    "objectiveText": "...",
    "alignmentLabel": "...",
    "alignmentSummary": "..."
  },
  "profile": {
    "inattentionSummary": "...",
    "impulsivitySummary": "..."
  },
  "dailyImpact": {
    "level": 4,
    "label": "지속적 소모가 큰 편",
    "empathy": "..."
  },
  "sections": {
    "summary": "...",
    "strength": "...",
    "watchout": "..."
  },
  "bridge": {
    "cta": "나만의 맞춤 훈련 설계하기"
  }
}
```

값 설명:

- `schemaVersion`: 현재 report 생성 규칙 버전
- `severity`: `낮음`, `중간`, `높음`
- `scores`: 레이더 그래프용 보조 점수
- `hero`: 첫 블록 요약
- `crossCheck`: 주관적 보고와 객관적 수행 비교
- `profile`: 부주의/충동성 상세 설명
- `dailyImpact`: DSM-5 기반 일상 피로도
- `sections.strength`: 보호 요인과 강점
- `sections.watchout`: 한계와 추가 평가 권고
- `bridge.cta`: plan으로 넘어가는 버튼 문구

해석 규칙:

- report 본문은 저장된 JSON 수치만 바탕으로 생성됩니다.
- 반응성 테스트 상세 값이 저장되어 있으면 그 값을 문장에 인용합니다.
- 반응성 테스트 상세 값이 없으면 `score` 수준 중심으로만 설명합니다.
- plan 문장과 달리 report는 임의 확장 서술을 하지 않도록 설계되어 있습니다.

## plan

저장 예시:

```json
"plan": {
  "suggestions": [
    "...",
    "...",
    "..."
  ],
  "chat": [
    {
      "role": "assistant",
      "text": "..."
    }
  ]
}
```

값 설명:

- `suggestions`: 기본 실행 제안 3개
- `chat`: plan 조정 대화 로그

해석:

- 기본 suggestions는 서버에서 정한 기본 텍스트를 우선 사용합니다.
- 사용자가 생활 패턴에 맞춘 조정을 요청하면 Gemini가 채팅 답변을 생성합니다.

## 종합적으로 보는 방법

1. ASRS
   - 주관적 자기보고에서 현재 어려움 강도를 먼저 봅니다.
   - `positive_count`, `attention_score`, `hyperactivity_score`를 중심으로 봅니다.

2. DSM-5
   - 증상 수와 subtype을 봅니다.
   - `inattention_true_count`, `hyperactivity_true_count`, `contextual_true_count`, `subtype`를 중심으로 봅니다.

3. 반응성 테스트
   - 실제 수행 기반의 객관 보조 지표로 봅니다.
   - score만 있는 경우와 상세 오류율이 있는 경우를 구분해서 해석합니다.

4. report
   - report는 저장된 JSON을 다시 읽어 주관/객관 지표를 한 화면으로 구조화합니다.

## 주의

- 이 앱의 결과는 진단이 아니라 선별 및 참고용입니다.
- 실제 해석은 지속 기간, 발현 시기, 다환경성, 기능 손상, 감별 진단 요소를 함께 봐야 합니다.
- JSON 수치는 해석을 위한 요약값이며, 단독으로 확정 판단을 내리기 위한 값은 아닙니다.
