# 질문과 해석

이 문서는 현재 저장되는 JSON 값이 무엇을 뜻하는지와, 화면에서 보이는 ASRS, DSM-5, 반응성 테스트 결과를 어떻게 해석하는지를 정리한 문서입니다.

## 저장 JSON 원칙

- 저장 파일은 문항 전문과 장문 로그를 넣지 않고, 점수와 핵심 집계값만 저장합니다.
- 화면 진행 중에는 메모리에서 현재 질문과 응답 상태를 사용하고, 파일에는 요약된 결과만 남깁니다.
- 따라서 `JSON 보기`에서 보이는 값은 최종 저장 기준의 간단한 구조입니다.

## ASRS

저장 예시:

```json
"asrs": {
  "responses": [1, 2, 3, 2, 4, 1],
  "attention_score": 8,
  "attention_max": 16,
  "hyperactivity_score": 5,
  "hyperactivity_max": 8,
  "total_score": 13,
  "total_max": 24,
  "positive_count": 4
}
```

값 설명:

- `responses`: 6개 문항의 원점수 배열
- `attention_score`: 1~4번 문항 합계
- `attention_max`: 1~4번 문항의 최대 가능 점수
- `hyperactivity_score`: 5~6번 문항 합계
- `hyperactivity_max`: 5~6번 문항의 최대 가능 점수
- `total_score`: 6문항 전체 합계
- `total_max`: 전체 최대 가능 점수
- `positive_count`: ASRS 기준상 유의미 문항 수

해석 규칙:

- 1~3번은 `2점 이상`이면 유의미
- 4~6번은 `3점 이상`이면 유의미
- `positive_count >= 4`면 선별 신호가 비교적 높은 편으로 봅니다
- `attention_score`가 높을수록 집중 유지, 시작 지연, 마감 관리 쪽 어려움 가능성이 커집니다
- `hyperactivity_score`가 높을수록 안절부절함, 끼어들기, 충동적 반응 가능성이 커집니다

## DSM-5

저장 예시:

```json
"dsm5": {
  "responses": [true, false, true, true, false, true, true, false, false, true, false, true, false, false, true, true, false, false, true, true, true, false, true],
  "inattention_true_count": 5,
  "hyperactivity_true_count": 4,
  "contextual_true_count": 4,
  "total_true_count": 9,
  "subtype": "판정 보류"
}
```

문항 구성:

- 1~9번: 부주의
- 10~18번: 과잉행동/충동성
- 19~23번: 추가 확인 문항
  - 6개월 이상 지속 여부
  - 12세 이전 일부 증상 여부
  - 2개 이상 환경에서의 반복 여부
  - 기능 저하 여부
  - 다른 요인만으로 설명되지 않는지 여부

값 설명:

- `responses`: DSM-5 전체 문항의 Yes/No 배열
- `inattention_true_count`: 부주의 9문항 중 Yes 개수
- `hyperactivity_true_count`: 과잉행동/충동성 9문항 중 Yes 개수
- `contextual_true_count`: 추가 확인 5문항 중 Yes 개수
- `total_true_count`: 부주의 + 과잉행동/충동성 Yes 총합
- `subtype`: 현재 규칙으로 계산한 유형 문자열

해석 규칙:

- `inattention_true_count >= 6` and `hyperactivity_true_count < 6`
  - `부주의형 가능성`
- `hyperactivity_true_count >= 6` and `inattention_true_count < 6`
  - `과잉행동/충동형 가능성`
- 두 값이 모두 `6 이상`
  - `복합형 가능성`
- 두 값이 모두 `6 미만`
  - `무증상 범위`
- 아직 전체 DSM-5 문항 응답이 끝나지 않으면
  - `판정 보류`

추가 확인 문항 해석:

- `contextual_true_count`는 유형 점수 자체를 만들지는 않지만, DSM-5 해석의 신뢰도를 보조합니다
- 값이 높을수록 지속성, 조기 발현, 다환경성, 기능 손상, 감별 필요성 조건이 더 많이 맞는다는 뜻입니다
- 특히 2개 이상 환경, 기능 저하, 다른 요인만으로 설명되지 않음은 실제 선별 해석에서 중요합니다

## 반응성 테스트

저장 예시:

```json
"game": {
  "status": "completed",
  "currentTestIndex": 2,
  "currentTestKey": "balance_hold",
  "tests": {
    "signal_detection": { "score": 81 },
    "go_nogo": { "score": 67 },
    "balance_hold": { "score": 72 }
  },
  "summary": {
    "inattention_signal_score": 81,
    "impulsivity_signal_score": 67,
    "activity_signal_score": 72
  }
}
```

값 설명:

- `status`: `pending`, `running`, `completed`
- `currentTestIndex`: 현재 진행 중인 테스트 순번
- `currentTestKey`: 현재 테스트 키
- `tests.signal_detection.score`: 지속주의/표적 탐지 점수
- `tests.go_nogo.score`: 반응 억제/충동 조절 점수
- `tests.balance_hold.score`: 활동성/자기조절 점수
- `summary.inattention_signal_score`: `signal_detection.score`와 동일한 의미
- `summary.impulsivity_signal_score`: `go_nogo.score`와 동일한 의미
- `summary.activity_signal_score`: `balance_hold.score`와 동일한 의미

해석 규칙:

- 점수는 `0~100` 범위에 가깝게 계산됩니다
- 점수가 높을수록 해당 기능이 더 안정적으로 수행된 것입니다
- 점수가 낮을수록 해당 영역에서 변동성, 오류, 조절 어려움 신호가 컸다는 뜻입니다

세부 의미:

- `signal_detection.score`
  - 파란 별 반응 유지, 누락 오류, 오반응, 반응시간 변동성을 반영
  - 낮으면 부주의 신호가 상대적으로 큽니다
- `go_nogo.score`
  - 멈춰야 할 순간의 제동 실패, 늦은 정지, 불필요한 정지 등을 반영
  - 낮으면 충동성 또는 억제 조절의 어려움 신호가 상대적으로 큽니다
- `balance_hold.score`
  - 흔들림 크기, 변동성, 큰 움직임 빈도를 반영
  - 낮으면 활동성/자기조절의 어려움 신호가 상대적으로 큽니다

## 종합적으로 보는 방법

### 1. ASRS

- 주관적 자기보고에서 현재 어려움 강도를 먼저 봅니다
- `positive_count`, `attention_score`, `hyperactivity_score`를 중심으로 봅니다

### 2. DSM-5

- 증상 수와 유형 분류를 봅니다
- `inattention_true_count`, `hyperactivity_true_count`, `subtype`를 중심으로 봅니다
- `contextual_true_count`는 해석을 보강하는 보조 조건입니다

### 3. 반응성 테스트

- 실제 수행 기반의 객관 보조 지표로 봅니다
- 주관 보고와 완전히 같지 않아도 되며, 감별 참고용입니다

## 예시 해석 패턴

- ASRS 높음 + DSM 부주의형 가능성 + signal_detection 낮음
  - 주의 유지와 실행 기능 쪽 어려움이 비교적 일관되게 나타난 패턴으로 해석할 수 있습니다

- ASRS 과잉행동/충동성 높음 + DSM 과잉행동/충동형 가능성 + go_nogo 낮음
  - 충동 반응과 억제 조절 쪽 신호가 상대적으로 두드러진 패턴으로 볼 수 있습니다

- DSM 복합형 가능성 + 반응성 테스트 전반 저하
  - 자기보고와 수행 지표 모두에서 여러 영역의 조절 부담이 함께 나타나는 패턴으로 볼 수 있습니다

- ASRS와 DSM은 높지만 반응성 테스트는 비교적 양호
  - 환경 적응, 검사 상황 집중, 보상 전략 사용 가능성을 함께 고려할 수 있습니다

- ASRS는 낮지만 DSM 추가 확인과 반응성 테스트가 좋지 않음
  - 자기인식 차이, 상황 특이성, 피로/수면 문제, 감정 상태 등의 영향을 함께 확인할 필요가 있습니다

## 주의

- 이 앱의 결과는 진단이 아니라 선별 및 참고용입니다
- 실제 해석은 지속 기간, 발현 시기, 다환경성, 기능 손상, 감별 진단 요소를 함께 봐야 합니다
- JSON 수치는 해석을 위한 요약값이며, 단독으로 확정 판단을 내리기 위한 값은 아닙니다
